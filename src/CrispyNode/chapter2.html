Node.js Fundamentals: Chapter 2

Working with Modules and NPM

In the previous chapter, we covered the basics of Node.js, including how to write and run simple scripts. In this chapter, we'll dive deeper into Node.js by looking at modules and NPM (Node Package Manager), two powerful tools that can help us write more complex applications.

Modules

In Node.js, a module is a reusable piece of code that encapsulates related functionality. Think of it as a package of code that can be used in different parts of your application. Modules help keep your code organized and modular, making it easier to maintain and update over time.

To create a module in Node.js, you simply define a JavaScript file and export the functions or variables you want to make available to other parts of your application. For example, let's say we want to create a module that generates random numbers. We could define a JavaScript file like this:


            // random.js
            function generateRandomNumber() {
              return Math.random();
            }
            
            module.exports = {
              generateRandomNumber: generateRandomNumber
            }

Here, we've defined a function called `generateRandomNumber` and exported it using the `module.exports` object. This makes the function available to other parts of our application.

To use a module in Node.js, we simply require it in our code using the `require` function. For example, if we wanted to use our `random` module from above, we could do this:


            // app.js
            const random = require('./random');
            
            console.log(random.generateRandomNumber());

Here, we're requiring our `random` module and calling the `generateRandomNumber` function to generate a random number.

Node.js comes with a number of built-in modules that you can use in your applications, such as `http` and `fs` (file system). You can also create your own modules and publish them to NPM for others to use.

NPM

NPM (Node Package Manager) is a package manager for Node.js that makes it easy to install, update, and manage third-party modules in your applications. NPM is included with Node.js, so you don't need to install it separately.

To install a module using NPM, you simply run the `npm install` command followed by the name of the module you want to install. For example, to install the `express` module, you would run:


            npm install express

This will download and install the `express` module and all of its dependencies in your project's `node_modules` directory.

Node.js Fundamentals - Chapter 2: Building Web Applications with Express

In the previous chapter, we learned how to install Node.js and write a simple "Hello, World!" program. In this chapter, we'll take it a step further and learn how to build web applications using Node.js and the Express framework.

What is Express?

Express is a fast, minimalist, and flexible web application framework for Node.js. It provides a robust set of features to develop web and mobile applications. Some of the core features of Express include:

  • Routing
  • Middleware
  • Templating engines
  • Error handling
  • Static file serving

With these features, Express simplifies the process of building web applications with Node.js.

Installing Express

To get started with Express, we need to install it. We can do this using npm, which is the package manager for Node.js. Open your terminal and navigate to your project directory.

$ cd my-project

Next, we need to install Express. To do this, we run the following command:

$ npm install express

This will download and install the `express` module and all of its dependencies in your project's `node_modules` directory.

Creating a Simple Web Application

Now that we have Express installed, let's create a simple web application. First, we need to create an instance of the `express` module:

const express = require('express');
          const app = express();

The `express` function returns an instance of an Express application, which we have stored in the `app` variable. We can now use this instance to create routes and configure middleware.

Let's create a route that responds with "Hello, World!" when a user visits the homepage:

app.get('/', (req, res) => {
            res.send('Hello, World!');
          });

The `app.get()` method creates a route for the GET HTTP method. The first argument is the path for the route (in this case, the homepage), and the second argument is a callback function that is executed when the route is matched. The callback function takes two parameters: the request object (`req`) and the response object (`res`). In this example, we're using the `res.send()` method to send the "Hello, World!" message to the user's web browser.

Now let's start our server and listen for incoming requests:

app.listen(3000, () => {
            console.log('Server started on port 3000');
          });

The `app.listen()` method starts the server and listens for incoming requests on the specified port (in this case, port 3000). When the server starts, we log a message to the console to confirm that it's running.

That's it! We've created a simple web application using Express.

Conclusion

In this chapter, we learned about the fundamentals of Node.js and how to use the Express framework to build a simple web application. We covered topics such as creating a basic server, routing requests, handling query parameters, serving static files, and using middleware.

We also briefly touched on some important Node.js concepts such as the event loop, callbacks, and the `require` function.

With this knowledge, you should have a solid foundation to build more complex web applications using Node.js and its vast ecosystem of modules and frameworks.
