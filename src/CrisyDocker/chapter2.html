Chapter 2: Working with Containers

In the previous chapter, we learned about the basics of Docker and how it can help us in developing and deploying applications. In this chapter, we will dive deeper into the world of Docker containers and learn how to work with them.

What are Containers?

Containers are a lightweight way to package software with all its dependencies and configuration. Containers enable developers to isolate applications from the underlying system and run them consistently across different environments.

A container is an instance of a Docker image. An image is a read-only template that contains instructions for creating a Docker container. Each container runs in its own environment and has its own file system, network interfaces, and processes.

Creating a Docker Container

To create a Docker container, we need to start with an existing Docker image. Docker Hub is a public repository that hosts a large collection of Docker images. We can search for an image using the docker search command:

$ docker search nginx

This will display a list of all the available Docker images related to nginx.

Once we have found the image we want to use, we can download it to our system using the docker pull command:

$ docker pull nginx

This will download the latest version of the nginx image to our system. We can then create a container using the docker run command:

$ docker run nginx

This will start a new container using the nginx image. By default, the container will run in the foreground and log all its output to the console. To stop the container, press Ctrl+C.

Viewing Running Containers

To view all the running Docker containers on our system, we can use the docker ps command:

$ docker ps

This will display a list of all the running containers along with their metadata such as container ID, image, command, status, and ports.

Stopping and Removing Containers

To stop a running container, we can use the docker stop command followed by the container ID:

$ docker stop <container-id>

This will gracefully stop the container by sending a SIGTERM signal to the main process inside the container. If the container does not stop within a certain time, Docker will forcefully stop it by sending a SIGKILL signal.

To remove a stopped container, we can use the docker rm command followed by the container ID:

$ docker rm <container-id>

This will permanently remove the container and its file system.

Conclusion

In this chapter, we learned about Docker containers and how to create, view, stop, and remove them. In the next chapter, we will explore how to interact with containers and perform common tasks such as networking and storage management.
