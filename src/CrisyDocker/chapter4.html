Chapter 4: Docker Networking

Metaphor: Just as a city's infrastructure relies on a network of roads, bridges, and tunnels to allow people and goods to move between different locations, Docker containers rely on networking to communicate with each other and with external systems.

Joke: Why did the Docker container need a friend? Because it was feeling isolated!

In this chapter, we'll explore the networking capabilities of Docker and learn how to set up and manage networks for our containers.

What is Docker Networking?

When Docker containers are created, they are assigned a unique IP address on a virtual network created by Docker. This network allows containers to communicate with each other and with external systems, such as the host machine or other containers running on the same network.

Docker provides several options for configuring networking for containers, including:

  • Bridge networking - the default networking mode for Docker, which allows containers to communicate with each other and with the host machine
  • Host networking - where containers share the host's network stack, making them appear as if they were running directly on the host
  • Overlay networking - which allows containers to communicate with each other across different host machines
  • Macvlan networking - which allows containers to have their own unique MAC addresses and appear as if they were physical machines on the network

Creating Docker Networks

Docker allows us to create our own custom networks for our containers, which can provide greater control and security. To create a network, we can use the docker network create command:

docker network create my-network

This will create a new network named my-network with the default bridge driver. We can also specify the driver using the --driver option:

docker network create --driver overlay my-overlay-network

This will create a new network named my-overlay-network using the overlay driver.

Connecting Containers to Networks

Once we have created a network, we can connect containers to it using the --network option when creating or running a container:

docker run --name my-container --network my-network my-image

This will create a new container named my-container and connect it to the my-network network. We can also connect an existing container to a network using the docker network connect command:

docker network connect my-network my-container

This will connect the existing container named my-container to the my-network network.

Conclusion

Networking is an important aspect of Docker containers, allowing them to communicate with each other and with external systems. In this chapter, we learned about the different networking options provided by Docker and how to create and connect containers to custom networks.
